# COO(사업 총괄) 팀 — 팀 개요

## 1역할 구성(team-of-one / 사람 직속)

| 역할 | 책무 | 추천 `--role` | `speciality` 예 | 상세 |
|------|------|--------------|-----------------|------|
| **COO** | 위임 판단·부문 감시(machine)·KPI 집계(machine)·경영 보고(machine)·부문 횡단 조율 | `manager` | `coo` | `coo/coo/` |

**본 템플릿의 근본적 특징: COO는 Anima 조직의 사령탑이며, 「부하 관리」와 「사람에 대한 경영 상황 집약 보고」의 2축을 담당한다.**

다른 팀 템플릿(개발·법무·재무 등)이 특정 업무 도메인을 갖는 데 비해, COO는 「조직 운영 자체」가 도메인이다. 특정 업무를 직접 수행하는 것이 아니라, 부문 간 조율·위임 판단·성과 모니터링·사람에 대한 보고가 주된 업무이다.

역할 디렉터리에 `injection.template.md`(injection.md 템플릿), `machine.md`(조직 분석·KPI 집계 파이프라인), `checklist.md`(품질 체크리스트)가 있다.

> 기본 원칙 상세: `team-design/guide.md`

## 운용 규모

- 현행: 솔로 운용(1명)
- 스팬 오브 컨트롤: 직속 {max_direct_reports}명까지(기본 기준: 10명)
- 스케일 기준: 직속 부하가 {max_direct_reports}를 초과하거나, 부문 수가 증가해 1명으로 감시가 어려운 경우
- 스케일 후 구성안: COO + VP(부문군 분담 총괄)

## 운용 모드

### Governance mode(부하 관리)

```
org_dashboard / audit_subordinate로 상황 파악
  → machine으로 조직 분석(컨텍스트 절약)
    → 이상 감지 시:
      직속 부하에게 delegate_task / send_message로 지시
      중대 안건은 call_human으로 사람에게 에스컬레이션
    → 정상 시:
      다음 리뷰까지 대기
```

### Reporting mode(경영 보고)

```
activity_log / org_dashboard에서 통계 정보 수집
  → machine으로 KPI 집계 + 보고서 드래프트 작성
    → COO가 검증·판단 코멘트 추가
      → call_human으로 사람에게 보고
      → 필요 시 비서에게 PDF화 의뢰
```

### Coordination mode(부문 횡단 조율)

```
부문 간 과제 감지(heartbeat / 부하의 보고)
  → COO가 판단:
    재무 관련 → 재무 리더와 send_message로 조율
    법무 관련 → 법무 리더와 send_message로 조율
    전체 방침 → 사람에게 에스컬레이션
    실행 태스크 → 적절한 부문 리더에게 delegate_task
```

## 통신 경로

```
사람 ←── call_human ──── COO (Anima)
                           ├── 직속 부하(부문 리더군)
                           │     ├── {개발 리더} → 개발 팀
                           │     ├── {CS 리더} → CS 팀
                           │     └── ...
                           ├── 탑레벨 동료
                           │     ├── Secretary(비서: PDF 위탁·정보 공유)
                           │     ├── CFO / 재무(재무 보고·예산 승인)
                           │     └── Legal Director(법무: 컴플라이언스 확인)
                           └── 내부 감사(독립: 감사 보고 수령만)
```

| 방향 | 수단 | 용도 |
|------|------|------|
| 사람 → COO | 채팅(Web UI) | 경영 지시, 방침 확인 |
| COO → 사람 | `call_human` | 경영 보고, 승인 요청, 에스컬레이션 |
| COO → 직속 부하 | `delegate_task` / `send_message` | 태스크 위임, 진행 확인, 지시 |
| 직속 부하 → COO | `send_message`(Inbox) | 진행 보고, 에스컬레이션 |
| COO → 탑레벨 동료 | `send_message` | 부문 횡단 조율, 정보 공유 |
| COO → 비서 | `send_message` | PDF화 의뢰, 정보 공유 |
| COO → 전체 | `post_channel` | 조직 공지(Board) |

## 위임 판단 기준

| 영향도 \ 긴급도 | 높음(당일 대응) | 중간(수일 이내) | 낮음(주 단위) |
|----------------|---------------|---------------|-------------|
| **높음**(경영·재무·법적 영향) | 사람에게 에스컬레이션 | COO가 판단, 탑 멤버(재무·법무)와 교섭 | COO가 판단, 해당 부문에 위임 |
| **중간**(부문 횡단·복수 팀) | COO가 직접 조율 | 부문 리더에 위임 | 부문 리더에 위임 |
| **낮음**(단일 부문 내) | 부문 리더에 위임 | 부문 리더에 위임 | 다음 리뷰에서 확인 |

**판단 원칙:**
- 사람에게 영향이 미치는 판단은 반드시 call_human으로 에스컬레이션
- 재무·법무가 관련된 안건은 탑레벨 멤버와 합의 필수
- 단일 부문 내 안건은 부문 리더에 위임(건너뛰기 지시 금지)

## 스팬 오브 컨트롤 관리

| 파라미터 | 기본값 | 비고 |
|---------|--------|------|
| max_direct_reports | 10 | 직속 부하 상한. 초과 시 사람에게 조직 재편 제안 |

- heartbeat에서 직속 부하 수를 자동 확인
- 초과 시: `call_human`으로 사람에게 조직 재편(VP 추가 등) 제안
- 일시적 초과(신규 멤버 온보딩 기간 등)는 허용하나 상시화는 피함

## 라우팅 규칙

직속 부하의 하위 멤버에게 직접 지시하지 않는다(NEVER). 반드시 부문 리더를 경유한다.

| 안건 카테고리 | 라우팅 대상 | 비고 |
|-------------|-----------|------|
| 개발·기술 | {개발 리더명} | 엔지니어 직접 지시 금지 |
| CS·고객 대응 | {CS 리더명} | |
| 인프라·모니터링 | {인프라 리더명} | |
| {카테고리} | {리더명} | 도입 시 설정 |

## 탑레벨 동료와의 협력

| 협력 대상 | 협력 패턴 | 용도 |
|----------|----------|------|
| Secretary(비서) | COO → 비서: PDF화·외부 전송 의뢰 / 비서 → COO: 트리아지 결과 배분 | 사람 대상 문서는 비서 경유로 정형·배포 |
| CFO / 재무 리더 | COO ↔ 재무: 예산 승인·결산 리뷰·투자 판단 | 재무 영향 판단은 재무와 합의 필수 |
| Legal Director / 법무 리더 | COO ↔ 법무: 컴플라이언스 확인·계약 리뷰 | 법적 리스크 판단은 법무와 합의 필수 |
| 내부 감사 | 감사 → COO: 감사 보고 수령(COO의 감사 개입 금지) | 독립성 보장 |

## 스케일링

| 규모 | 구성 | 비고 |
|------|------|------|
| 솔로 | COO 1명(본 템플릿) | 직속 {max_direct_reports}명 이내 |
| COO + VP | COO + VP 1~2명 | VP가 부문군 분담 총괄. COO는 전체 방침·사람 보고에 집중 |

## cron 설정 예시

도입 시 참고 예시. 조직 규모·업무 사이클에 맞춰 설정한다.

| 태스크 | schedule 예 | type | 개요 |
|--------|------------|------|------|
| 아침 활동 다이제스트 | `0 9 * * *` | llm | 전 Anima의 최근 24h 활동을 집약, call_human으로 사람에게 보고 |
| 일일 부하 감사 | `0 15 * * *` | llm | org_dashboard + audit_subordinate로 전 직속 부하 상황 확인, knowledge/에 기록 |
| 이사회 개회 | `0 9 * * *` | llm | #board에 안건 투고 |
| 이사회 요약 | `0 11 * * *` | llm | #board의 논의를 정리, 필요 시 사람에게 에스컬레이션 |
| 밤 우선순위 정리 | `30 21 * * *` | llm | 미완료 태스크 트리아지, 다음 날 우선사항 정리 |
| 주간 조직 회고 | `0 17 * * 5` | llm | episodes/에서 주간 요약 작성, knowledge/에 기록 |
