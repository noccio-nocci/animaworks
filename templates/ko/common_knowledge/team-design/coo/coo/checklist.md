# COO — 품질 체크리스트

machine 사용 여부와 관계없이, COO가 산출물·판단의 품질을 보장하기 위한 체크리스트.

---

## A. 위임 품질

- [ ] 적절한 부문 리더에 위임하고 있는가(건너뛰기 지시 없는가)
- [ ] 위임 시 충분한 정보를 전달했는가(목적·기한·성공 기준)
- [ ] 위임 후 추적이 이루어지고 있는가(STALE / 미응답 감지)
- [ ] 팔로업 타이밍이 적절한가(너무 이르지도 늦지도 않은가)
- [ ] 사람 지시(source: human) 태스크를 최우선으로 처리하고 있는가

---

## B. 감사 품질

- [ ] org_dashboard / audit_subordinate를 정기적으로 실행하고 있는가
- [ ] STALE 태스크를 놓치지 않았는가
- [ ] 에러·장애 감지 시 적절히 에스컬레이션했는가
- [ ] 부하가 2시간 이상 tool_use 없을 때 상황 확인했는가
- [ ] 감사 결과를 knowledge/에 기록했는가

---

## C. 보고 품질

- [ ] 사람에 대한 정기 보고 누락이 없는가(일일 다이제스트, 주간 회고 등)
- [ ] KPI·분석 결과에 COO 자신의 판단 코멘트가 있는가(machine 출력 그대로가 아닌가)
- [ ] 액션 아이템이 명확한가(누가·무엇을·언제까지)
- [ ] 에스컬레이션이 필요한 사항을 놓치지 않았는가
- [ ] 보고 포맷이 일관적인가

---

## D. 부문 횡단 조율

- [ ] 재무·법무와 합의가 필요한 안건에서 합의를 얻었는가
- [ ] 방치된 부문 간 갈등이 없는가
- [ ] 복수 부문에 걸치는 과제를 적절히 조율·라우팅했는가
- [ ] Board(#board)를 활용해 조직 횡단 정보 공유를 하고 있는가

---

## E. 스팬 오브 컨트롤

- [ ] 직속 부하 수가 {max_direct_reports} 이내인가
- [ ] 초과 시 사람에게 조직 재편을 보고했는가
- [ ] 각 부문 리더의 부하가 편중되지 않았는가(특정 리더에 과집중 없는가)
