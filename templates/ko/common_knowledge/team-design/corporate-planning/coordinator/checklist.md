# Strategy Coordinator — 품질 체크리스트

machine 사용 여부와 관계없이 Strategy Coordinator가 검증 품질을 보증하기 위한 체크리스트.

---

## A. 검증 품질

- [ ] 검증 계획서를 스스로 작성했는가(machine에게 설계시키지 않았는가)
- [ ] machine 출력을 메타 검증했는가(그대로 전달하지 않았는가)
- [ ] Critical / Warning 각 지적에 구체적 근거가 있는가
- [ ] 지적 내용이 사실에 기초하는가(추측이나 착각이 아닌가)
- [ ] 오검출(False Positive)을 제거했는가
- [ ] 누락(False Negative)을 보완했는가

---

## B. 가정 챌린지

- [ ] Strategist 전제 가정을 전건 나열했는가
- [ ] 각 가정에 대한 반증적 검증이 포함되어 있는가
- [ ] 낙관 편향을 검출하는가(「잘될 것」전제 항목)
- [ ] 최악 시나리오가 구체적이고 현실적인가
- [ ] 가정이 무너졌을 때의 영향도를 평가하는가

---

## C. KPI 정합

- [ ] KPI 실태와 전략 목표의 괴리를 확인했는가
- [ ] 데이터 소스가 정확한가(부서 제공 데이터 vs 추정값 구분)
- [ ] KPI 측정 방법·기간이 일관되는가
- [ ] 괴리가 큰 KPI에 대응 방침이 제시되어 있는가

---

## D. Tracker 관리

- [ ] Strategic Initiative Tracker 전건 status가 확인되었는가
- [ ] 정체 이니셔티브(1개월 이상 스테이지 변화 없음)가 보고되었는가
- [ ] silent drop이 0인가(전회 존재 항목이 전건 남아 있는가)
- [ ] 신규 이니셔티브가 Tracker에 등록되어 있는가

---

## E. 조치 전달

- [ ] Strategist 승인 후 각 부서에 조치 전달이 빠짐없이 이루어졌는가
- [ ] 전달 내용에 필요한 정보(목적·스코프·기한·기대 성과)가 포함되어 있는가
- [ ] 후속 스케줄이 설정되었는가
- [ ] 각 부서로부터 피드백 수령 체제가 갖춰져 있는가

---

## F. 메타 검증 품질

### machine 검증 결과의 검증

- [ ] 중요도 판정(Critical / Warning / Info)이 타당한가
- [ ] 가정 챌린지 논리 전개가 정확한가
- [ ] KPI 대조 수치가 정확한가
- [ ] Tracker 대조에 누락이 없는가

### 피드백 품질

- [ ] Critical 지적에 구체적 근거와 권장 수정이 포함되어 있는가
- [ ] 건설적 톤인가(문제 지적 + 개선 제안)
- [ ] `verification-report.md`의 `status: approved`를 확인한 뒤 Strategist에게 송부했는가
