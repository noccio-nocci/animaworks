# Infrastructure Monitor — 품질 체크리스트

machine을 사용하지 않으므로, 모니터링 실행·보고 품질·이상 감지의 정확성을 보증하기 위한 체크리스트이다.

---

## A. 모니터링 실행

- [ ] cron으로 정의된 전체 모니터링 항목을 실행했는가
- [ ] 실행 실패한 항목이 없는가(실패한 경우 이상으로 보고했는가)
- [ ] 모니터링 명령어의 타임아웃이나 접속 오류를 감지했는가

---

## B. 보고 품질

- [ ] 전체 점검 항목에 값과 임계값이 기록되어 있는가(「문제 없음」만으로 처리하지 않았는가)
- [ ] 정기 모니터링 보고 템플릿 포맷에 준거하는가
- [ ] 요약의 전체 상태(✅ / ⚠️ / 🔴)가 모니터링 결과와 정합하는가

---

## C. 이전 미해결 항목

- [ ] 이전 미해결 항목이 전건 이월되었는가(silent drop 제로)
- [ ] 각 미해결 항목의 상태가 갱신되었는가(해소·계속·악화)
- [ ] 해소된 항목에 해소 근거가 기재되어 있는가

---

## D. 이상 감지

- [ ] 임계값 초과를 놓치지 않았는가
- [ ] 중요도 판정(INFO / WARNING / CRITICAL)이 에스컬레이션 임계값 테이블에 준거하는가
- [ ] WARNING / CRITICAL 감지 시 이상 감지 보고를 Director에게 즉시 전송했는가

---

## E. 초기 대응

- [ ] 복구 절차를 실행한 경우, 결과가 이상 감지 보고에 포함되어 있는가
- [ ] 절차 불명의 이상에 자기 판단으로 대응하지 않았는가(Director에게 보고만이 원칙)
- [ ] 초기 대응의 실시 여부와 결과를 명기했는가
