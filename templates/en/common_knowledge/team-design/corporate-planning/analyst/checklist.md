# Business Analyst — quality checklist

Checklist for Business Analyst to protect research and analysis quality with or without machine.

---

## A. Source quality

- [ ] Every item has a source (URL, date)
- [ ] Each source has a confidence rating (High/Med/Low)
- [ ] Primary sources (official releases, statistics, government) are used where possible
- [ ] For secondary sources (articles, blogs, etc.), primary sources were sought where feasible
- [ ] “Unverified” / “needs more research” is explicit where applicable

---

## B. Research scope

- [ ] Covers the Strategist’s request scope
- [ ] No unjustified scope creep
- [ ] Research order followed requested priorities
- [ ] If items were excluded, rationale is stated

---

## C. Analysis quality

- [ ] Facts vs interpretation are separated
- [ ] Machine output was validated (no speculative gap-filling)
- [ ] Conclusions are backed by data and sources
- [ ] Conflicting data, if any, is presented on both sides
- [ ] Statistics state unit, period, and population

---

## D. Freshness

- [ ] Retrieval date on every data point
- [ ] Notes on data 6+ months old
- [ ] Market changes (regulation, new entrants, etc.) reflected where relevant
- [ ] Competitor intel based on latest public information

---

## E. Output format

### `market-analysis.md`

- [ ] Follows template format
- [ ] Source table complete
- [ ] Self-check section filled
- [ ] `status: approved` before delivery to Strategist

### `competitive-report.md`

- [ ] Comparison matrix included
- [ ] Per-company analysis cites sources
- [ ] Positioning analysis included
- [ ] `status: approved` before delivery to Strategist
