# CS Lead — Quality Checklist

Quality checklist for CS Lead to ensure deliverable quality, with or without machine usage.

---

## A. Onboarding (Phase A)

- [ ] All sections of cs-handoff.md (customer overview, sales process summary, agreements, unresolved items, communication characteristics) are reflected in the onboarding plan
- [ ] Agreements/requests are translated into concrete execution steps
- [ ] Unresolved items have response plans included
- [ ] Communication characteristics are considered in the approach
- [ ] Confirmed `status: approved` before delegating to Support
- [ ] Completion criteria and deadlines set for each step

---

## B. Health Analysis (Phase B)

- [ ] All customers in Health Tracker have updated scores (zero silent drops)
- [ ] Retention actions drafted for Yellow/Red customers (Phase C linkage)
- [ ] Score changes (vs. previous) confirmed with recorded reasons
- [ ] Churn prediction rationale is explicitly stated (avoid judgment by speculation alone)
- [ ] Critical (Red) customers escalated upward

---

## C. Response Quality (Phase C — machine output verification)

- [ ] Customized appropriately for the customer's situation
- [ ] Tone is appropriate (considering cs-handoff.md communication characteristics)
- [ ] No compliance issues
- [ ] Proposals are feasible (no over-promising)
- [ ] Factual information (customer name, contact name, dates) is accurate

---

## D. VoC Aggregation (Phase D)

- [ ] Inquiry trends accurately reflected
- [ ] Improvement proposals backed by evidence (VoC data)
- [ ] Impact scope and priority stated for each proposal
- [ ] COO report properly formatted
- [ ] Changes from previous VoC report noted

---

## E. Tracker Management

- [ ] All Health Tracker items have updated statuses
- [ ] No missing registrations for new customers (post-onboarding)
- [ ] Zero silent drops (no items deleted since last review)
- [ ] Next actions set for all items

---

## F. Feedback Handling

### From Support

- [ ] All reports from Support reviewed
- [ ] Reflected in Health Tracker
- [ ] Identified and executed items requiring upward reporting

### Delegation quality

- [ ] Confirmed `status: approved` on onboarding-plan.md
- [ ] Support has access to necessary information (cs-handoff.md, communication approach)
- [ ] Deadline set
