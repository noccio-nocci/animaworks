# COO（事業統括）チーム — チーム概要

## 1ロール構成（team-of-one / 人間直属）

| ロール | 責務 | 推奨 `--role` | `speciality` 例 | 詳細 |
|--------|------|--------------|-----------------|------|
| **COO** | 委任判断・部門監視（machine）・KPI集計（machine）・経営報告（machine）・部門横断調整 | `manager` | `coo` | `coo/coo/` |

**本テンプレートの根本的な特徴: COO は Anima 組織の司令塔であり、「部下の管理」と「人間への経営状況の集約レポート」の2軸を担う。**

他のチームテンプレート（開発・法務・財務等）が特定の業務ドメインを持つのに対し、COOは「組織運営そのもの」がドメインである。特定業務を自ら実行するのではなく、部門間の調整・委任判断・パフォーマンス監視・人間への報告が主務となる。

ロールディレクトリに `injection.template.md`（injection.md 雛形）、`machine.md`（組織分析・KPI集計パイプライン）、`checklist.md`（品質チェックリスト）がある。

> 基本原則の詳細: `team-design/guide.md`

## 運用規模

- 現行: ソロ運用（1名）
- スパンオブコントロール: 直属 {max_direct_reports} 名まで（デフォルト目安: 10名）
- スケール基準: 直属部下が {max_direct_reports} を超えた場合、または部門数が増大し1名では監視が追いつかない場合
- スケール後の構成案: COO + VP（部門群を分担統括）

## 運用モード

### Governance mode（部下管理）

```
org_dashboard / audit_subordinate で状況把握
  → machine で組織分析（コンテキスト節約）
    → 異常検知時:
      直属部下に delegate_task / send_message で指示
      重大案件は call_human で人間にエスカレーション
    → 正常時:
      次回レビューまで待機
```

### Reporting mode（経営報告）

```
activity_log / org_dashboard から統計情報を収集
  → machine で KPI集計 + レポートドラフト作成
    → COO が検証・判断コメント追加
      → call_human で人間に報告
      → 必要に応じて秘書に PDF 化を依頼
```

### Coordination mode（部門横断調整）

```
部門間の課題を検知（heartbeat / 部下からの報告）
  → COO が判断:
    財務関連 → 財務リーダーと send_message で調整
    法務関連 → 法務リーダーと send_message で調整
    全体方針 → 人間にエスカレーション
    実行タスク → 適切な部門リーダーに delegate_task
```

## 通信経路

```
人間 ←── call_human ──── COO (Anima)
                           ├── 直属部下（部門リーダー群）
                           │     ├── {開発リーダー} → 開発チーム
                           │     ├── {CSリーダー} → CSチーム
                           │     └── ...
                           ├── トップレベル同僚
                           │     ├── Secretary（秘書: PDF委託・情報共有）
                           │     ├── CFO / 財務（財務報告・予算承認）
                           │     └── Legal Director（法務: コンプライアンス確認）
                           └── 内部監査（独立: 監査報告の受領のみ）
```

| 方向 | 手段 | 用途 |
|------|------|------|
| 人間 → COO | チャット（Web UI） | 経営指示、方針確認 |
| COO → 人間 | `call_human` | 経営報告、承認依頼、エスカレーション |
| COO → 直属部下 | `delegate_task` / `send_message` | タスク委任、進捗確認、指示 |
| 直属部下 → COO | `send_message`（Inbox） | 進捗報告、エスカレーション |
| COO → トップレベル同僚 | `send_message` | 部門横断調整、情報共有 |
| COO → 秘書 | `send_message` | PDF化依頼、情報共有 |
| COO → 全体 | `post_channel` | 組織アナウンス（Board） |

## 委任判断基準

| 影響度 \ 緊急度 | 高（即日対応） | 中（数日以内） | 低（週単位） |
|----------------|-------------|-------------|------------|
| **高**（経営・財務・法的影響） | 人間にエスカレーション | COO が判断し、トップメンバー（財務・法務）と交渉 | COO が判断し、該当部門に委任 |
| **中**（部門横断・複数チーム） | COO が直接調整 | 該当部門リーダーに委任 | 該当部門リーダーに委任 |
| **低**（単一部門内） | 該当部門リーダーに委任 | 該当部門リーダーに委任 | 次回レビューで確認 |

**判断原則:**
- 人間に影響が及ぶ判断は必ず call_human でエスカレーションする
- 財務・法務が関わる案件はトップレベルメンバーと合意を取る
- 単一部門で完結する案件は部門リーダーに委任する（飛び越え指示禁止）

## スパンオブコントロール管理

| パラメータ | デフォルト | 備考 |
|-----------|-----------|------|
| max_direct_reports | 10 | 直属部下の上限数。超過時は人間に組織再編を提案 |

- heartbeat で直属部下数を自動チェックする
- 超過時: `call_human` で人間に組織再編（VP 追加等）を提案する
- 一時的な超過（新メンバーのオンボーディング期間等）は許容するが、常態化は避ける

## ルーティングルール

直属部下の配下メンバーには直接指示を出さない（NEVER）。必ず部門リーダー経由で指示する。

| 案件カテゴリ | ルーティング先 | 備考 |
|------------|-------------|------|
| 開発・技術 | {開発リーダー名} | エンジニアへの直接指示禁止 |
| CS・顧客対応 | {CSリーダー名} | |
| インフラ・監視 | {インフラリーダー名} | |
| {カテゴリ} | {リーダー名} | 導入時に設定 |

## トップレベル同僚との連携

| 連携先 | 連携パターン | 用途 |
|--------|-----------|------|
| Secretary（秘書） | COO → 秘書: PDF化依頼、外部送信依頼 / 秘書 → COO: トリアージ結果の分配 | 人間向けドキュメントは秘書経由で整形・配信 |
| CFO / 財務リーダー | COO ↔ 財務: 予算承認・決算レビュー・投資判断 | 財務影響のある判断は財務と合意必須 |
| Legal Director / 法務リーダー | COO ↔ 法務: コンプライアンス確認・契約レビュー | 法的リスクのある判断は法務と合意必須 |
| 内部監査 | 監査 → COO: 監査報告の受領（COOから監査への介入は禁止） | 独立性担保 |

## スケーリング

| 規模 | 構成 | 備考 |
|------|------|------|
| ソロ | COO 1名（本テンプレート） | 直属部下 {max_direct_reports} 名以内 |
| COO + VP | COO + VP 1〜2名 | VP が部門群を分担統括。COO は全体方針・人間報告に専念 |

## cron 設定例

以下は導入時の参考例。組織規模・業務サイクルに合わせて設定する。

| タスク | schedule 例 | type | 概要 |
|--------|------------|------|------|
| 朝の活動ダイジェスト | `0 9 * * *` | llm | 全Animaの直近24h活動を集約し、call_human で人間に報告 |
| 日次部下監査 | `0 15 * * *` | llm | org_dashboard + audit_subordinate で全直属部下の状況を確認し、knowledge/ に記録 |
| 取締役会開会 | `0 9 * * *` | llm | #board に議題投稿 |
| 取締役会サマリー | `0 11 * * *` | llm | #board の議論をまとめ、必要に応じて人間にエスカレーション |
| 夜の優先度整理 | `30 21 * * *` | llm | 未完了タスクのトリアージ、翌日の優先事項を整理 |
| 週次組織振り返り | `0 17 * * 5` | llm | episodes/ から週次サマリーを作成し、knowledge/ に記録 |

| — | — | COO（経営報告） | **COO固有。他チームに対応なし** |
