# COO — 品質チェックリスト

machine 使用有無に関わらず、COO が成果物・判断の品質を担保するためのチェックリスト。

---

## A. 委任品質

- [ ] 適切な部門リーダーに委任しているか（飛び越え指示がないか）
- [ ] 委任時に十分な情報を渡しているか（目的・期限・成功基準）
- [ ] 委任後の追跡が行われているか（STALE / 未応答の検知）
- [ ] フォローアップのタイミングが適切か（早すぎず遅すぎず）
- [ ] 人間からの指示（source: human）を最優先で処理しているか

---

## B. 監査品質

- [ ] org_dashboard / audit_subordinate を定期的に実行しているか
- [ ] STALE タスクを見落としていないか
- [ ] エラー・障害を検知した場合、適切にエスカレーションしているか
- [ ] 部下が 2h 以上 tool_use なしの場合、状況確認しているか
- [ ] 監査結果を knowledge/ に記録しているか

---

## C. 報告品質

- [ ] 人間への定期報告が漏れていないか（日次ダイジェスト、週次振り返り等）
- [ ] KPI・分析結果に COO 自身の判断コメントがあるか（machine 出力そのままでないか）
- [ ] アクションアイテムが明確か（誰が・何を・いつまでに）
- [ ] エスカレーションが必要な事項を見落としていないか
- [ ] 報告のフォーマットが一貫しているか

---

## D. 部門横断調整

- [ ] 財務・法務との合意を取るべき案件で合意を取っているか
- [ ] 部門間コンフリクトの放置がないか
- [ ] 複数部門にまたがる課題を適切に調整・ルーティングしているか
- [ ] Board（#board）を活用して組織横断の情報共有を行っているか

---

## E. スパンオブコントロール

- [ ] 直属部下数が {max_direct_reports} 以内か
- [ ] 超過時に人間に組織再編を報告しているか
- [ ] 各部門リーダーの負荷が偏っていないか（特定リーダーに過集中していないか）
