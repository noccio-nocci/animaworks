# CS Lead — 品質チェックリスト

machine 使用有無に関わらず、CS Lead が成果物の品質を担保するためのチェックリスト。

---

## A. オンボーディング（Phase A）

- [ ] cs-handoff.md の全セクション（顧客概要、営業プロセス要約、合意事項、未解決事項、コミュニケーション特性）がオンボーディング計画に反映されているか
- [ ] 合意事項・要望が具体的な実行ステップに落ちているか
- [ ] 未解決事項に対する対応方針が含まれているか
- [ ] コミュニケーション特性が考慮された対応方針になっているか
- [ ] Support への委譲前に `status: approved` を確認したか
- [ ] 完了条件と期限が各ステップに設定されているか

---

## B. ヘルス分析（Phase B）

- [ ] Health Tracker の全顧客のスコアを更新したか（silent drop ゼロ）
- [ ] Yellow/Red の顧客に対してリテンション施策を策定したか（Phase C 連携）
- [ ] スコア変動（前回比）を確認し、変動理由を記録したか
- [ ] チャーン予測の根拠が明記されているか（推測のみでの判定を避ける）
- [ ] Critical（Red）の顧客について上位にエスカレーションしたか

---

## C. 対応品質（Phase C — machine 出力検証）

- [ ] 顧客状況に適したカスタマイズがされているか
- [ ] トーンが適切か（cs-handoff.md のコミュニケーション特性を考慮）
- [ ] コンプライアンス上の問題がないか
- [ ] 提案内容が実現可能か（過大な約束をしていないか）
- [ ] 顧客名・担当者名・日付等の事実情報に誤りがないか

---

## D. VoC 集約（Phase D）

- [ ] 問い合わせ傾向が正確に反映されているか
- [ ] 改善提案に根拠（VoC データ）があるか
- [ ] 改善提案の影響範囲と優先度が明記されているか
- [ ] COO 報告の体裁が整っているか
- [ ] 前回の VoC レポートからの変化点が記載されているか

---

## E. Tracker 管理

- [ ] Health Tracker の全件ステータスが更新されているか
- [ ] 新規顧客（オンボーディング完了後）の登録漏れがないか
- [ ] silent drop がゼロであるか（前回から削除された項目がないか）
- [ ] 次回アクションが全件に設定されているか

---

## F. フィードバック対応

### Support からの報告

- [ ] Support からの報告を全件確認したか
- [ ] Health Tracker に反映したか
- [ ] 上位報告が必要な件を判定し、実施したか

### 委譲時の品質

- [ ] onboarding-plan.md の `status: approved` を確認したか
- [ ] Support が実行に必要な情報（cs-handoff.md、コミュニケーション方針）がアクセス可能か
- [ ] 期限（deadline）を設定したか
