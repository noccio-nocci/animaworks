# Corporate Strategist — 品質チェックリスト

machine 使用有無に関わらず、Corporate Strategist が成果物の品質を担保するためのチェックリスト。

---

## A. 事業環境分析

- [ ] Analyst の market-analysis.md が情報ソース（URL, 日付）付きか
- [ ] PEST/SWOT 等のフレームワークが分析目的に対して適切に適用されているか
- [ ] 「勝てる領域」の抽出に明確な根拠があるか
- [ ] 機会/脅威の優先順位付けが論理的か
- [ ] 内部データと外部データの整合性が取れているか
- [ ] 楽観的な仮定に根拠があるか（Coordinator の検証で指摘されうる箇所を事前確認）

---

## B. 戦略提案書

- [ ] OKR との整合性が明示されているか
- [ ] 施策の実行可能性（リソース・期間・技術的制約）が検討されているか
- [ ] リソース配分の妥当性（過度な集中・分散がないか）
- [ ] リスク分析が含まれているか（影響度 × 発生可能性）
- [ ] 各施策に具体的な成功指標（KPI）があるか
- [ ] 代替案が検討されているか

---

## C. 進捗分析

- [ ] Strategic Initiative Tracker の全件ステータスが更新されているか
- [ ] 停滞イニシアチブ（1ヶ月以上ステージ変化なし）に対処済みか
- [ ] silent drop がゼロか（前回存在した項目が全件残っているか）
- [ ] KPI 達成見込みの評価が含まれているか
- [ ] 「中止」判定のイニシアチブに理由が明記されているか

---

## D. 委譲時の品質

### Analyst への調査依頼前に

- [ ] 調査の目的・スコープ・期限が明記されているか
- [ ] 優先度（何を最初に調べるべきか）が示されているか
- [ ] 期待する出力形式が明確か

### Coordinator への検証依頼前に

- [ ] strategy-report.md の `status: reviewed` を確認したか
- [ ] strategic-plan.md（分析計画書）が添付されているか
- [ ] Coordinator が検証に必要な情報（市場分析レポート・内部データ）がアクセス可能か

---

## E. フィードバック対応

- [ ] Coordinator からのフィードバックを全件確認したか
- [ ] Critical 指摘に対して全体再分析を実施したか（Analyst に再調査依頼済みか）
- [ ] Warning 指摘に対して差分確認・修正を実施したか
- [ ] Strategic Initiative Tracker の最終ステータスを更新したか
- [ ] 最終レポートに全フィードバックの対応結果が反映されているか
