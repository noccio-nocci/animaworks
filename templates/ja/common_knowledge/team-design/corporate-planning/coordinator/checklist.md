# Strategy Coordinator — 品質チェックリスト

machine 使用有無に関わらず、Strategy Coordinator が検証品質を担保するためのチェックリスト。

---

## A. 検証品質

- [ ] 検証計画書を自分で書いたか（machine に設計させていないか）
- [ ] machine の出力をメタ検証したか（そのまま転送していないか）
- [ ] Critical / Warning の各指摘に具体的な根拠があるか
- [ ] 指摘内容が事実に基づいているか（推測や思い込みでないか）
- [ ] 誤検出（False Positive）を除去したか
- [ ] 見落とし（False Negative）を補完したか

---

## B. 仮定チャレンジ

- [ ] Strategist の前提仮定を全件リストアップしたか
- [ ] 各仮定に対する反証的検証が含まれているか
- [ ] 楽観バイアスを検出しているか（「うまくいく」前提の項目）
- [ ] 最悪シナリオが具体的かつ現実的か
- [ ] 仮定が崩れた場合の影響度を評価しているか

---

## C. KPI 整合

- [ ] KPI 実態と戦略目標の乖離を確認したか
- [ ] データソースが正確か（部門提供データ vs 推定値の区別）
- [ ] KPI の測定方法・期間が一貫しているか
- [ ] 乖離が大きい KPI に対応方針が示されているか

---

## D. Tracker 管理

- [ ] Strategic Initiative Tracker の全件ステータスが確認されているか
- [ ] 停滞イニシアチブ（1ヶ月以上ステージ変化なし）が報告されているか
- [ ] silent drop がゼロか（前回存在した項目が全件残っているか）
- [ ] 新規イニシアチブが Tracker に登録されているか

---

## E. 施策伝達

- [ ] Strategist の承認後、各部門への施策伝達が漏れなく行われたか
- [ ] 伝達内容に必要な情報（目的・スコープ・期限・期待成果）が含まれているか
- [ ] フォローアップのスケジュールが設定されたか
- [ ] 各部門からのフィードバック受領体制が整っているか

---

## F. メタ検証の品質

### machine 検証結果の検証

- [ ] 重要度の判定（Critical / Warning / Info）が妥当か
- [ ] 仮定チャレンジの論理展開が正確か
- [ ] KPI 照合の数値が正確か
- [ ] Tracker 照合に漏れがないか

### フィードバックの品質

- [ ] Critical 指摘に具体的な根拠と推奨修正が含まれているか
- [ ] 建設的なトーンになっているか（問題の指摘 + 改善提案）
- [ ] `verification-report.md` の `status: approved` を確認してから Strategist に送付したか
